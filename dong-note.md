# Flyway 

## 왜 도입해야하는가
- 데이터베이스와 코드 사이의 간격때문에 발생하는 촌극 : 
- 디비 스키마 변경을 모두 수작업으로 하고 변경이력을 관리하지 않으면
```text
100% 신뢰할수 있는 유효성 검증이 불가능
- JPA 를 사용하지 않으면 
휴먼에러의 문제성
DB 스키마를 코드로 관리하는것이 바로 이거다
```


## DB 스키마를 코드로 관리하는 이유
- DB 스키마 변경에 대한 작업이력이 남는다
- 코드 변경처럼 스키마 변경또한 이력이 남아야 한다!!
